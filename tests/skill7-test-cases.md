## 测试方法说明

本文档为**人工评审型测试用例**。使用方法：

1. 将 `SKILL-v7.md` 的完整内容作为 System Prompt 加载到 Claude（或其他 LLM）
2. 按照下方每个测试用例的"测试输入"发送消息
3. 对照"验证标准"逐条检查输出
4. 在"评审结果"栏记录 ✅ PASS / ❌ FAIL / ⚠️ PARTIAL
5. 建议每个测试用例运行 2-3 次以检验输出稳定性

### TC-12: Application Synthesis 三维度（偶像揭露 + 具体行动 + 最坏情况）

| 项目 | 内容 |
|------|------|
| **验证标准 — 偶像揭露（Idol Exposure）** | |
| ☐ 12a | 精准识别场景背后参与者真正害怕失去的"偶像" |
| ☐ 12b | 偶像命名具体（不是泛泛的"骄傲"或"恐惧"） |
| ☐ 12c | 揭露**多层偶像**（表层偶像 → 深层偶像），不止一层 |
| ☐ 12d | 示例：表层="专业形象"，深层="掌控别人对我的评价" |
| **验证标准 — 具体行动（Concrete Action）** | |
| ☐ 12e | 提供**明天就能在工作中说出的具体话术**（exact words） |
| ☐ 12f | 提供**立即可执行的具体行动**（不仅是"调整心态"） |
| ☐ 12g | 至少包含一个可以直接使用的对话示例 |
| ☐ 12h | 行动建议用上司/同事听得懂的语言框架（如团队士气、管理风险） |
| **验证标准 — 最坏情况（Worst-Case Scenario）** | |
| ☐ 12i | 预设参与者坚持圣经原则但**没有奇迹降临** |
| ☐ 12j | 描述真实的世俗代价（被解雇、被边缘化、失去客户、晋升受阻等） |
| ☐ 12k | ❌ 不出现童话式结局（如"老板被你感动了"、"失去一个客户但奇迹般得到更大的"） |
| ☐ 12l | 在最坏情况中提供信仰如何成为根基的论述 |
| **评审结果** | |

5. **TC-21** — 验证 Self-Check 清单

### API 自动化测试提示

如需通过 Anthropic API 自动化测试，建议：

```python
# 伪代码示例
import anthropic

client = anthropic.Anthropic()

# 加载 SKILL-v7.md 作为 system prompt
with open("SKILL-v7.md", "r") as f:
    system_prompt = f.read()

# 测试对话流
messages = [
    {"role": "user", "content": "Hello"},
    # 检查是否返回语言选择菜单 → TC-01
    {"role": "user", "content": "2"},
    # 检查是否返回中文 Greeting → TC-02
    {"role": "user", "content": "马可福音第2章"},
    # 检查完整输出 → TC-03 到 TC-21
]

# 逐轮发送并收集输出
for msg in messages:
    response = client.messages.create(
        model="claude-sonnet-4-20250514",
        max_tokens=8192,
        system=system_prompt,
        messages=[msg]
    )
    print(response.content[0].text)
    # 人工或自动化检查验证标准
```
